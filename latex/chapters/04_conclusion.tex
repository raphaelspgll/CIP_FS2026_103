\section{Conclusion}

\subsection{Limitations}

\begin{itemize}[itemsep=0pt,topsep=4pt]
  \item The feature set is limited to price-derived technical indicators; sentiment data, on-chain metrics, and macro variables are not included.
  \item CoinGecko does not expose intra-day open prices, so \texttt{price\_change\_pct} is always recomputed as a day-over-day return rather than an intra-day figure.
  \item The Decision Tree is not tuned; hyperparameter search (e.g.\ max depth, min samples) could improve performance and reduce overfitting.
\end{itemize}

\subsection{Summary}

This project demonstrates a reproducible end-to-end pipeline for cryptocurrency price-direction prediction, from web scraping through preprocessing, feature engineering, and classification.
Although the achieved accuracy is modest, the pipeline is methodologically sound — chronological splits, lag-correct features, and transparent imputation flags ensure that the results reflect genuine out-of-sample performance.

Future work could extend the feature set with sentiment signals, experiment with ensemble methods, or explore longer prediction horizons to see whether predictability increases over multi-day windows.
